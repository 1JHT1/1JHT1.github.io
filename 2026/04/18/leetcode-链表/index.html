138—随机链表的复制

给你一个长度为 n 的链表，每个节点包含一个额外增加的随机指针 random ，该指针可以指向链表中的任何节点或空节点。构造这个链表的 深拷贝。 深拷贝应该正好由 n 个 全新 节点组成，其中每个新节点的值都设为其对应的原节点的值。新节点的 next 指针和 random 指针也都应指向复制链表中的新节点，并使原链表和复制链表中的这些指针能够表示相同的链表状态。复制链表中的指针都不应指向原链表中的节点 。例如，如果原链表中有 X 和 Y 两个节点，其中 X.random --> Y 。那么在复制链表中对应的两个节点 x 和 y ，同样有 x.random --> y 。返回复制链表的头节点。用一个由 n 个节点组成的链表来表示输入/输出中的链表。每个节点用一个 [val, random_index] 表示：val：一个表示 Node.val 的整数。random_index：随机指针指向的节点索引（范围从 0 到 n-1）；如果不指向任何节点，则为 null 。你的代码 只 接受原链表的头节点 head 作为传入参数。

如果是普通的链表，直接按照遍历顺序创建链表即可，但由于这里的节点涉及到随机指针，在我们拷贝节点的时候，当前节点的随机节点指针可能还未创建，故利用回溯的思路，让每个节点的拷贝操作相互独立，对于当前节点，我们首先拷贝，然后进行后继节点和随机指针指向节点的拷贝，拷贝完成就将创建的新节点指针返回，故完成了当前节点的两个指针的赋值。

用HashMap记录每一个节点对应的新节点的创建情况，遍历链表过程中，检查当前节点的后继和随机指针创建的节点清空，如果其中一个没有创建，则立刻递归地创建！这里很巧妙的去递归的实现未创建节点然后进行拷贝，当我们拷贝完成，会回溯到当前层继续进行拷贝完成当前节点的拷贝。

特别注意的是一个节点可能被多个其他节点指向，因此我们在递归的时候为了防止重复拷贝，先检查一遍当前节点是否被拷贝过，已经拷贝则直接在HashMap中返回拷贝后的节点指针。

回溯+HashMap，时间复杂度O(n)、空间复杂度O(n)：

下面用另一种想法来省去用HashMap的空间：

在我们复制前将原来链表的每一个节点都赋值一个在其后，例如A->B->C变成A->A’->B->B’->C->C’，这种情况下我们可以你直接获得拷贝节点的下一个随机节点理应的对应节点（由于保持了原来链表的特性），故每一个拷贝节点的随机下一个节点在原链表已经锁定，但是随机下一个节点可能为null

当完成随机的后，我们只要将这个链表按原节点和拷贝节点拆分，遍历一次，需要注意最后一个拷贝节点的后继节点为null

迭代+节点拆分：时间复杂度O(n)、空间复杂度O(1)

146—LRU缓存

请你设计并实现一个满足 LRU (最近最少使用) 缓存 约束的数据结构。实现 LRUCache 类：

  • LRUCache(int capacity) 以 正整数 作为容量 capacity 初始化 LRU 缓存
  • int get(int key) 如果关键字 key 存在于缓存中，则返回关键字的值，否则返回 -1 。
  • void put(int key, int value) 如果关键字 key 已经存在，则变更其数据值 value ；如果不存在，则向缓存中插入该组 key-value 。如果插入操作导致关键字数量超过 capacity ，则应该 逐出 最久未使用的关键字。

函数 get 和 put 必须以 O(1) 的平均时间复杂度运行。
