StringBuilder

StringBuilder是对动态数组（char[]）的封装，用于高效处理可变字符串。 是 Java 中用于创建可变字符串的类，位于 java.lang 包中。它提供了一个可变的字符序列，适合在需要频繁修改字符串内容（例如拼接、插入、删除）的场景下使用，比直接使用 String 更加高效。

1. 为什么需要 StringBuilder？

  • String 是不可变的：每次对 String 进行拼接、替换等操作，都会创建一个新的 String 对象。如果频繁操作，会产生大量临时对象，降低性能并增加垃圾回收压力。
  • StringBuilder 是可变的：它内部维护一个可变的字符数组，修改操作直接在原数组上进行，避免了创建中间对象。

2. 与 StringBuffer 的区别

特性 StringBuilder StringBuffer
线程安全 否 是（方法用 synchronized 修饰）
性能 更高（无同步开销） 较低
使用场景 单线程环境 多线程环境

在实际开发中，绝大多数情况使用 StringBuilder 即可，除非明确需要线程安全。

5.扩容机制

当内部字符数组容量不足时，StringBuilder 会自动扩容：新容量 = 旧容量 * 2 + 2，然后将原数组内容复制到新数组中。频繁扩容会影响性能，因此如果能预估最终字符串长度，建议在构造时指定合适的初始容量。

6. 小结

  • 用途：解决频繁修改字符串时的性能问题。
  • 特点：可变、非线程安全、效率高。
  • 注意：不需要线程安全时优先使用 StringBuilder，否则用 StringBuffer。
  • 重要方法：append、insert、delete、replace、reverse、toString。
