35—搜索插入位置

给定一个排序数组和一个目标值，在数组中找到目标值，并返回其索引。如果目标值不存在于数组中，返回它将会被按顺序插入的位置。

请必须使用时间复杂度为 O(log n) 的算法。

二分查找最后目标值不在数组的时候，此时的left一定是大于target的，且最接近target，因此只需要输出left即可！

34—在排序数组中查找元素的第一个和最后一个位置

给你一个按照非递减顺序排列的整数数组 nums，和一个目标值 target。请你找出给定目标值在数组中的开始位置和结束位置。如果数组中不存在目标值 target，返回 [-1, -1]。

你必须设计并实现时间复杂度为 O(log n) 的算法解决此问题。

分析：当时间复杂度低于O(n)时说明我们不能遍历数组,故采取双指针，但是下面是我第一次提供的算法，时间复杂度最坏时O(n)!,也就是说当数组的值全是目标值的时候，我们的扩展会遍历到所有元素，此时复杂度是O(n),而且边界条件上一开始我写错了，扩展边界的做法，一定要先判断扩展后的结果是否再数组索引范围内，再进行条件的判断以及对应逻辑的书写。永远遵循：先检查索引边界，再访问数组元素。

当然我们可以采用二分查找去寻找边界此时的算法时间复杂度正好是O(logn)

首先编写一个找目标值第一次出现的位置算法作为left，然后用 //int end = lowerBound(nums, target + 1) - 1;这段代码找的事比target大1的第一个出现的位置，那这个位置再减去1就说明是target最后出现的位置。

这种将 target 和 target+1 的查找结果相减来得到 target 的出现区间，在解决“统计有序数组中某个值的出现次数”等问题中非常常见。

33—搜索旋转排序数组

整数数组 nums 按升序排列，数组中的值 互不相同 。在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 向左旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 下标 3 上向左旋转后可能变为 [4,5,6,7,0,1,2] 。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

分析：对于旋转数组而言我们可以通过二分查找获取旋转的位置下标，实际上就是最小值的下标！因为旋转后可能会导致最小值跑到旋转位置k去，这个时候只需要找到最小值下标就能得到旋转的位置k了。

再对二个有序区间进行二分操作即可得到对应的target下标！

当然如果想只使用一次二分的话，我们就需要意识到这个问题，我们的二分法一定会将原来的半有序分成一半有序和一半至多有序(很可能无序),所以我们只需要对在target所在区间二分即可，因为最差的情况是target落在无序的那个区间，但是我们持续二分一定会使得target落在有序区间，最差是直到最后一个元素是target！

只要意识到上面的特点，我们只需要将原来的二分查找的查找条件**尽可能的控制二分区间里有target就可以！**故需要考虑旋转数组的第一个元素值是够大于target，如果大于，那么可能在二分的第一个区间，小于说明可能在第二个区间，

153—寻找旋转排序数组中的最小值

已知一个长度为 n 的数组，预先按照升序排列，经由 1 到 n 次 旋转 后，得到输入数组。例如，原数组 nums = [0,1,2,4,5,6,7] 在变化后可能得到：

  • 若旋转 4 次，则可以得到 [4,5,6,7,0,1,2]
  • 若旋转 7 次，则可以得到 [0,1,2,4,5,6,7]

注意，数组 [a[0], a[1], a[2], ..., a[n-1]] 旋转一次 的结果为数组 [a[n-1], a[0], a[1], a[2], ..., a[n-2]] 。

给你一个元素值 互不相同 的数组 nums ，它原来是一个升序排列的数组，并按上述情形进行了多次旋转。请你找出并返回数组中的 最小元素 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

分析：这题实际就改进二分查找并获取最小值的索引或者说是旋转次数
